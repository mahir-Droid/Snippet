4.3 Create Search Cluster: R3


4.3.1 Description and Priority

A user logged in to the web platform can create a cluster where they can input multiple URLs to search and retrive data from. The type of data can be selected by the user form the categories: pdf, doc, text, non-html, all categories. The user then clicks a submit button to search the urls and fetch the data accordingly which is saved as a cluster to their account for them to seach later.


4.3.2 Stimulus/Response Sequence

Actor action System Response
1. A logged in user in their view cluster page clicks on Create Cluster button. 2. Prompts the first Create Cluster screen with input: Cluster Name and integer input(1-5): How many urls?
3. User inputs the name and number of clusters(1-5) and clicks submit button. 4. Prompts the second Create Cluster screen with muliple(according to user input) URL input with category selections each.
5. User inputs the urls and their categories respectively multiple times(according to their previous integer input). 6. The information is passed to the system and it fetches data from each URL according to their respective types and saves them in a cluster. The user is then redirected to the view cluster page where they can see the representaion of their newly created cluster.


4.3.3 Functional Requirements

User must be in web platform to execute this feature. This feature is unavailable in mobile platform. The user should input valid URLs to search. The validity of the URLs must be checked by the system. Upon wrong input, the system should redirect user back to the second Create Cluster page with error message: "Enter valid URL". In the first Create Cluster page, the input form of number of cluster should be in a dropdown box with stated numbers 1-5 to choose from.

The cluster should fetch and update data automatically at regular intervals to keep its data on par with the new updates of the URLs.
