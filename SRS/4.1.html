4.1 Login: R1


4.1.1 Description and Priority

Users are able to log in to their accounts using their credentials: username and password, upon visiting the application through the website or the mobile app.


4.1.2 Stimulus/Response Sequence

Actor action System Response
1. User visits the website or mobile app to their log in screen. 2. Prompts a user log in screen with username and password as input.
3. User inputs their credentials and clicks on login button 4. Login credentials are verified and user is prompted with their personal view cluster screen.


4.1.3 Functional Requirements

The user must input correct user credentials for the login page to correctly route to the personalized view cluster page. Upon wrong credential input the user is rerouted back to the login page with an error message: “Wrong username/password”. User should already have an account in the platform to be able to login.
