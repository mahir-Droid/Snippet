5. Other Nonfunctional Requirements


5.1 Performance Requirements

The search operation should run with an indexing system like Solr(refer to 1.5) and be optimized to improve search speed.


5.2 Safety Requirements

The search results that are exported by the user may contain many data that are not under free-use policy. Futhermore, the website that it was scraped from may have a citation and fair use policy. The user must be notified before exporting data to take the responsibility and research the respective website and/or company about their data usage policy.


5.3 Security Requirements

Users details must be stored in a secure database and will only be able to modify their own personal details. Their passwords must be stored in hash. Users will have access constraints so that only personal data can be accessed.


5.4 Software Quality Attributes

The system should have one backend system that both web platform and mobile platform can access. This should be done with an api that will connect the mobile platform sitting outside the backend system. Furthermore, crawling systems and searching systems should also sit outside the backend project connected through apis. This, even though complicates the initail code base, makes the project simpler to scale; such as incorporating new crawling systems, search indexing, integrating to new platforms.
